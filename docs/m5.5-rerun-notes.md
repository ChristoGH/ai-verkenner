Re-validation of the hub-dampened convergence signal on **real data**, after the M6 smoke
(`docs/m6-smoke-notes.md`) proved `/horizon` ranked ubiquitous hubs at the top. Run on **2026-06-20**.

- **Stores:** `docker compose up qdrant neo4j` (live). **Embedder:** real `sentence-transformers`
  (`BAAI/bge-small-en-v1.5`, 384-dim). **LLM:** real Anthropic `claude-opus-4-8`.
- **Commands:** `python -m app.cli reindex` (recreated the Qdrant `items` collection at 384 via the
  M6 dim self-heal) → `python -m app.cli run`.
- **Feed set (bounded — flagged):** 6 independent working feeds (more than M6's 4):
  GitHub Blog, Simon Willison's Weblog, Qdrant Releases, arXiv cs.AI, arXiv cs.CL, arXiv cs.LG
  (`ARXIV_MAX_RESULTS=8`).
  - **Why not the full `sources.yaml`:** sizing showed **Hugging Face Blog returned 803 entries and
    Eugene Yan 210** (their feeds serve the full archive, not just recent) — together ~1,100 items,
    an unbounded LLM bill. `LlamaIndex Blog` 404s. Excluded HF/Eugene/LlamaIndex; flagged as a real
    finding (feed pagination — a per-source recency cap is worth adding later). The 6-feed set is a
    clear, cost-bounded superset of M6's working sources (~$10).

## Does /horizon now surface emerging signals instead of hubs? — **Yes. This is the headline.**

Same data, two rankings:

**Entity level — top by RAW distinct sources (the OLD signal) vs M5.5 dampened contribution:**


`'GitHub'` — **the entity that topped M6's `/horizon`** — now scores **0.00**: it has 6 developments

## Evidence reconciled (M6 finding #3) — ✅

For the real signal, the card's `contributing_sources` now equals the **driving entity's** distinct
sources, so the claim and the evidence agree:

```
why: convergence: 'Calibration' across 2 independent sources in 2 developments
contributing_sources: ['arXiv cs.LG', 'arXiv cs.AI']   (matches why? True)
```

Non-signal items (convergence 0) carry an **empty** `why` (no false claim) and fall back to their
own coverage sources — no more "says 4 sources, lists 1" mismatch.

## What still looks wrong / flagged

1. **"Independent sources" are arXiv *categories* here.** The strongest convergence in this corpus
   comes from arXiv **cross-listing** (cs.AI ∩ cs.LG count as two sources, but it's one publisher).
   The dampening is correct; the corpus just lacks much *cross-publisher* overlap. A feed set with
   independent outlets covering the same launches (the excluded labs/news feeds) would produce
   stronger, more meaningful signals. The 'Calibration' signal is plausible but modest.
2. **Feed pagination (HF 803 / Eugene 210).** Some RSS feeds serve their whole archive; a per-source
   recency cap (config) would let a full-`sources.yaml` run stay bounded. Out of M5.5 scope; noted.
3. **`person`-as-author still extracted** (e.g. an author entity), but the independence gate now
   neutralises single-feed authors automatically — no special-casing needed, as predicted.

## Verdict

The hub-dampening works on real data: the M6 hub ('GitHub') is suppressed to zero, a rare
cross-source/cross-development entity ('Calibration') rises to the top with matching evidence, and
semantic dedup now genuinely collapses cross-source coverage. `/horizon` surfaces *emerging* signals
rather than *loud* ones — the behaviour the "Weak Signal of the Week" output needs. The remaining
limitation is corpus breadth (true cross-publisher overlap), not the ranking logic.
