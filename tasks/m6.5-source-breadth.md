# Task M6.5 — Source breadth

**Status: M6.5 — IN REVIEW** (implemented on `feat/m6.5-source-breadth`; awaiting review gate)

An evidence-driven slice inserted after the **M5.5 re-run** (`docs/m5.5-rerun-notes.md`), which
proved the convergence signal ranks correctly but the corpus **lacks cross-publisher overlap** to
feed it — the strongest signal it could find was arXiv cross-listing (same publisher).

## Why

`/horizon` is starved of **independent cross-publisher sources**. This slice feeds the convergence
machine: implement the long-stubbed `github_*` intelligence fetchers (curated, per ADR 0002 — NOT a
crawler), bound archive-serving feeds so a full run is affordable, and broaden the registry with
reputable independent outlets that cover the same developments — then re-validate that real
cross-publisher convergence appears.

## Scope

1. **Per-source recency cap** (config, applied uniformly in ingestion, fail-safe):
   `SOURCE_MAX_AGE_DAYS` (skip items older than the window) + `SOURCE_MAX_ITEMS` (hard ceiling).
   Bounds archive feeds (Hugging Face served 803, Eugene Yan 210).
2. **The four curated `github_*` fetchers** (official GitHub API; watched orgs/users/topics/packages
   from `sources.yaml`; `GITHUB_TOKEN` required, skip-not-crash when absent; rate-limit aware; URLs
   preserved):
   - `github_new_repos` — recent repos from a watched org/user or topic (created in window).
   - `github_star_velocity` — **honest** delta vs the previous run's star snapshot (persisted to
     SQLite); first run seeds the baseline and emits nothing. Does NOT fake velocity from absolutes.
   - `github_advisories` — advisories affecting a watched package in the user's stack (Early Warning).
   - `github_changes` — recent breaking-change/deprecation issues/PRs by label in a watched repo.
3. **Broaden + clean `sources.yaml`** — fix the LlamaIndex 404 (→ releases), add independent
   reputable outlets (Ars Technica AI, VentureBeat AI, MIT Tech Review, The Register, HN), enable the
   `github_*` sources with concrete watched topics/packages/repos. Trust/enabled kept honest.

## Non-goals

- No `priority.py` / ranking / graph-schema changes, no M7 digest/feedback, no MCP, no post
  generator, no fuzzy entity resolution. **NOT a crawler** — watched-list + official API only.

## Acceptance criteria

- The recency cap bounds an oversized feed; `github_*` fetchers parse fixtures into RawItems with
  URLs; star-velocity emits nothing on the baseline run and the delta on the next; `github_*` skip
  cleanly without `GITHUB_TOKEN`; the registry validates the new entries.
- Re-run notes (`docs/m6.5-run-notes.md`) record: did the cap bound the big feeds? did `github_*`
  produce items? **and does `/horizon` now surface a cross-publisher convergence** (a development
  independent outlets covered) rather than arXiv cross-listing? — with before/after evidence + cost.
